CSS Variables & Custom Properties

CSS Custom Properties (commonly called "CSS variables") let you store values in named tokens and reuse them throughout your stylesheet. Change one variable and the update propagates everywhere — making design systems, themes, and maintenance dramatically easier.

Declaring and Using Variables

CSS — Declaring Custom Properties
/* Declare in :root for global access */ :root { --color-primary: #667eea; --color-text: #2c3e50; --spacing-md: 16px; --border-radius: 8px; --font-body: 'Inter', sans-serif; --shadow: 0 4px 6px rgba(0, 0, 0, 0.1); } /* Use with var() */ .button { background: var(--color-primary); padding: var(--spacing-md) calc(var(--spacing-md) * 2); border-radius: var(--border-radius); box-shadow: var(--shadow); font-family: var(--font-body); color: white; }
Naming Convention

Custom property names must start with two dashes (--) and are case-sensitive. Common naming patterns include --color-*, --space-*, --font-*, and --shadow-* — mirroring design token conventions from tools like Figma and Style Dictionary.

Fallback Values

CSS — Fallback with var()
/* Second argument to var() is the fallback */ .element { color: var(--color-primary, #667eea); } /* Fallbacks can chain */ .element { color: var(--user-theme-color, var(--color-primary, blue)); }

Scoped Variables

Variables can be declared on any element and will be inherited by its children — enabling component-level theming without touching global styles.

CSS — Component Scoping
/* Global default */ :root { --btn-bg: #667eea; --btn-color: white; } /* Override per variant — no need for extra classes */ .btn-danger { --btn-bg: #ef4444; } .btn-success { --btn-bg: #10b981; } .btn-outline { --btn-bg: transparent; --btn-color: #667eea; } /* Shared implementation */ .btn { background: var(--btn-bg); color: var(--btn-color); padding: 10px 20px; border-radius: 8px; border: 2px solid var(--btn-bg); }

Dark Mode Theming

CSS — Dark Mode via Variables
:root { --bg: white; --surface: #f9fafb; --text: #1f2937; --muted: #6b7280; --border: #e5e7eb; } /* Automatic dark mode */ @media (prefers-color-scheme: dark) { :root { --bg: #0f172a; --surface: #1e293b; --text: #e2e8f0; --muted: #94a3b8; --border: #334155; } } /* Manual toggle via data attribute */ [data-theme="dark"] { --bg: #0f172a; /* ... */ }

Using Variables with JavaScript

JavaScript — Reading & Writing CSS Variables
// Read a CSS variable const primary = getComputedStyle(document.documentElement) .getPropertyValue('--color-primary').trim(); // Write / update a CSS variable at runtime document.documentElement.style .setProperty('--color-primary', '#ef4444'); // Theme toggle document.getElementById('theme-btn').addEventListener('click', () => { document.documentElement.toggleAttribute('data-theme', 'dark'); });
CSS Variables vs Sass Variables

Sass variables ($primary: #667eea) are compiled away at build time — they cannot be changed at runtime. CSS custom properties live in the browser and can be read and updated by JavaScript, making them essential for theming, animations, and user preferences.

Wrapping Up

CSS Custom Properties are the foundation of every modern design system. Declaring tokens in :root, scoping overrides to components, and toggling themes with a single data-attribute means your entire site can be restyled with minimal code changes.
