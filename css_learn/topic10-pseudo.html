Pseudo-classes, Pseudo-elements & Advanced Selectors

Pseudo-classes and pseudo-elements let you style elements based on their state, position, or content — without adding extra HTML or JavaScript. Mastering them unlocks tooltips, decorative elements, zebra-striped tables, dynamic form feedback, and much more — entirely in CSS.

Pseudo-classes

A pseudo-class selects an element in a particular state. They use a single colon: :hover, :focus, etc.

User Interaction States

CSS — Interaction Pseudo-classes
a:link { color: #667eea; } /* unvisited link */ a:visited { color: #764ba2; } /* visited link */ a:hover { color: #4f46e5; } /* pointer over */ a:active { color: #ef4444; } /* being clicked */ /* :focus — keyboard / programmatic focus */ input:focus { border-color: #667eea; box-shadow: 0 0 0 3px rgba(102,126,234,0.3); } /* :focus-visible — only shows ring for keyboard navigation */ button:focus-visible { outline: 3px solid #667eea; outline-offset: 3px; } /* :disabled / :checked */ input:disabled { opacity: 0.5; cursor: not-allowed; } input:checked + label { color: #667eea; }

Structural Pseudo-classes

CSS — Structural Pseudo-classes
li:first-child { border-radius: 8px 8px 0 0; } li:last-child { border-radius: 0 0 8px 8px; } li:nth-child(odd) { background: #ede9fe; } li:nth-child(even) { background: #f3f4f6; } /* nth-child formula: An+B */ li:nth-child(3n) { font-weight: bold; } /* every 3rd */ li:nth-child(-n+3) { color: #ef4444; } /* first 3 only */ /* :only-child, :empty */ p:only-child { font-size: 1.2em; } div:empty { display: none; }
:nth-child(odd/even) — zebra stripe pattern
  • Row 1 — :first-child + :nth-child(odd)
  • Row 2 — :nth-child(even)
  • Row 3 — :nth-child(odd)
  • Row 4 — :nth-child(even)
  • Row 5 — :last-child + :nth-child(odd)

Form & Input States

CSS — Form Pseudo-classes
input:valid { border-color: #10b981; } input:invalid { border-color: #ef4444; } input:required{ border-left: 3px solid #f59e0b; } /* :placeholder-shown — no value typed yet */ input:placeholder-shown + label { transform: translateY(0); } /* :has() — style parent based on child state (modern CSS) */ label:has(input:checked) { background: #eef2ff; border-color: #667eea; }
:has() — label responds to its checkbox being checked

Pseudo-elements

Pseudo-elements use double colons (::) and create virtual elements in the DOM — no extra HTML needed.

CSS — Pseudo-elements
/* ::before and ::after insert generated content */ .blockquote::before { content: '\201C'; /* Unicode " character */ font-size: 3em; color: #667eea; position: absolute; left: 16px; } /* content can use attr() to pull HTML attributes */ [data-tip]::after { content: attr(data-tip); /* ... position as tooltip ... */ } /* ::first-line, ::first-letter */ article p::first-letter { font-size: 3em; float: left; line-height: 0.8; margin-right: 8px; } /* ::selection — custom text highlight */ ::selection { background: #667eea; color: white; } /* ::placeholder */ input::placeholder { color: #9ca3af; font-style: italic; }
Generated content via ::before and ::after — no HTML tags needed for the quotation mark or attribution.

Advanced Modern Selectors

CSS — :is(), :where(), :not(), :has()
/* :is() — match any in a list (takes highest specificity) */ :is(h1, h2, h3) { font-family: Georgia, serif; } /* :is() with descendant — DRY selectors */ :is(section, article) h2 { color: #667eea; } /* :where() — same as :is() but specificity is always 0 */ :where(h1, h2, h3) { line-height: 1.3; } /* :not() — exclude elements */ li:not(:last-child) { border-bottom: 1px solid #e5e7eb; } input:not([type="submit"]):not([type="checkbox"]) { border: 2px solid #d1d5db; } /* :has() — parent selector, selects based on children */ .card:has(img) { padding-top: 0; /* card has an image — remove top padding */ } figure:has(figcaption) { border: 1px solid #e5e7eb; }

The content Property is Required for ::before / ::after

Even if you don't want text, you must include content: "". Without it, the pseudo-element is not rendered. Use it with display: block or display: flex to create decorative shapes and overlays.

Don't Put Meaningful Content in ::before / ::after

Screen readers may or may not read generated content. Never use pseudo-elements to render text that is essential for understanding — use real HTML for that. Pseudo-elements are for decoration only.

Wrapping Up This Topic

Pseudo-classes give you state-aware styling — hover, focus, checked, nth-child — without any JavaScript. Pseudo-elements give you virtual DOM nodes for decorative elements, tooltips, and overlays. Together they let you write incredibly expressive styles with minimal HTML.
