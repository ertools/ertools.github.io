Transitions & Animations

Motion communicates meaning — it guides attention, confirms actions, and makes interfaces feel alive. CSS provides two ways to add motion: transitions (smooth changes between two states) and animations (multi-step sequences that can run continuously). Hover the examples on this page to see them live.

CSS Transitions

A transition smoothly animates a property change from one value to another — most commonly triggered by :hover, :focus, or a class added by JavaScript.

CSS — transition Syntax
/* transition: property duration timing-function delay */ .btn { background: #667eea; transition: background 0.3s ease; } .btn:hover { background: #4f46e5; } /* Multiple properties */ .card { transition: transform 0.3s ease, box-shadow 0.3s ease; } /* Transition all animatable properties */ .element { transition: all 0.25s ease; /* use sparingly — can be slow */ }

CSS Animations with @keyframes

Animations define a sequence of CSS states (keyframes) that the browser cycles through automatically.

CSS — @keyframes + animation
/* 1. Define the keyframes */ @keyframes fadeInUp { from { opacity: 0; transform: translateY(20px); } to { opacity: 1; transform: translateY(0); } } /* Multi-step with percentages */ @keyframes bounce { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-24px); } } /* 2. Apply the animation */ .hero-title { animation: fadeInUp 0.8s ease-out both; } /* name duration timing fill-mode */

Animation Properties

CSS — Individual Animation Properties
.spinner { animation-name: spin; animation-duration: 0.9s; animation-timing-function: linear; animation-iteration-count: infinite; /* or a number */ animation-direction: normal; /* alternate | reverse */ animation-fill-mode: both; /* forwards | backwards | both */ animation-delay: 0.2s; animation-play-state: running; /* paused | running */ }

The transform Property

Transforms are the most GPU-efficient way to animate movement, scale, and rotation. Combine them with transitions or animations for silky 60fps motion.

CSS — Common Transform Functions
.element { /* Translate (move) */ transform: translateX(20px); transform: translateY(-10px); transform: translate(20px, -10px); /* Scale */ transform: scale(1.1); /* 10% larger */ transform: scaleX(0.5); /* half width */ /* Rotate */ transform: rotate(45deg); /* Skew */ transform: skewX(15deg); /* Chain multiple transforms */ transform: translateY(-4px) scale(1.05) rotate(2deg); }
Animate transform and opacity — Never width or height

transform and opacity are handled by the GPU compositor and do not trigger layout recalculations, giving you smooth 60fps animations. Animating width, height, top, left, or margin forces the browser to repaint every frame — this is called "layout thrashing" and causes jank.

Respect prefers-reduced-motion

Always wrap decorative animations in a media query for users who request reduced motion for accessibility reasons:

@media (prefers-reduced-motion: reduce) { .anim { animation: none; transition: none; } }

Wrapping Up

CSS transitions handle two-state animations triggered by state changes; @keyframes handle continuous or multi-step sequences. Both are most performant when animating transform and opacity. Always respect prefers-reduced-motion.
