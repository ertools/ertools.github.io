Responsive Design & Media Queries

Responsive design means your website looks great and works correctly on every screen — from a 320px phone to a 4K monitor. The core tools are media queries, fluid units (%, vw, rem), and modern layout techniques like Flexbox and Grid that adapt naturally to available space.

The Viewport Meta Tag

This HTML tag is required for mobile-responsive behaviour. Without it, mobile browsers assume your page is desktop-width and zoom out.

HTML — Viewport Meta Tag
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

Media queries let you apply CSS rules only when certain conditions are met — most commonly a minimum or maximum viewport width.

CSS — Media Query Syntax
/* Apply styles only at 768px and wider */ @media (min-width: 768px) { .sidebar { display: block; } } /* Only at 480px and below */ @media (max-width: 480px) { .nav-links { display: none; } } /* Range with min and max */ @media (min-width: 600px) and (max-width: 900px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* Dark mode preference */ @media (prefers-color-scheme: dark) { body { background: #0f172a; color: #e2e8f0; } } /* Print styles */ @media print { .navbar, .sidebar { display: none; } }

Mobile-First vs Desktop-First

Mobile-First is the Recommended Approach

Write your base styles for mobile, then add complexity for larger screens with min-width queries. This results in smaller CSS sent to mobile devices (the most constrained) and aligns with how browsers process styles. Desktop-first uses max-width queries to strip features back for smaller screens — it often produces messier overrides.

CSS — Mobile-First Pattern
/* Base: mobile layout — 1 column */ .grid { display: grid; grid-template-columns: 1fr; gap: 16px; } /* Tablet (640px+): 2 columns */ @media (min-width: 640px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* Desktop (1024px+): 3 columns */ @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3, 1fr); } }

Design-Driven Breakpoints

Don't add breakpoints just because a standard pixel value exists — add them when your design breaks. Resize the browser slowly and add a breakpoint where the layout stops looking right.

CSS — Fluid Sizing Techniques
/* Fluid container */ .container { width: min(100%, 1200px); margin-inline: auto; padding-inline: clamp(16px, 5vw, 48px); } /* Fluid font size */ h1 { font-size: clamp(1.8rem, 5vw, 3.5rem); } /* Fluid image */ img { max-width: 100%; height: auto; display: block; }

Responsive Images

HTML — srcset for High-DPI & Responsive Images
<img src="photo-800.jpg" srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w" sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 33vw" alt="Responsive photo" loading="lazy" >

Container Queries (Modern CSS)

Container queries let an element respond to the size of its container rather than the viewport — perfect for reusable components.

CSS — Container Queries
/* 1. Declare the containment context */ .card-wrapper { container-type: inline-size; container-name: card; } /* 2. Style based on container width */ @container card (min-width: 400px) { .card { flex-direction: row; } }
Don't Forget Touch Targets

On mobile, interactive elements (buttons, links) should be at least 44×44px in size (Apple / WCAG guideline). Use padding to enlarge small elements without changing their visual appearance.

Wrapping Up

Responsive design is built on a mobile-first philosophy, media queries for layout pivots, fluid units for sizing, and native layout tools like Grid and Flexbox. Together, they ensure your site works beautifully on every screen without needing separate mobile and desktop codebases.
