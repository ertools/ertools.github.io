Display & Positioning

Two of the most important CSS properties are display — which controls how an element generates its box — and position — which controls how that box is placed in the document. Together they are the foundation of every layout on the web.

The display Property

ValueBehaviourExamples
blockTakes full width, starts on a new line<div>, <p>, <h1>
inlineFlows with text, ignores width/height<span>, <a>, <strong>
inline-blockFlows with text but respects width/heightIcon + label combos, nav items

CSS — Display Values
/* Turn a list into a horizontal navigation */ nav li { display: inline-block; padding: 8px 16px; } /* Hide an element completely */ .hidden { display: none; } /* Hide but keep space (use visibility instead) */ .invisible { visibility: hidden; } /* Control opacity without removing layout space */ .faded { opacity: 0.4; }

The position Property

static (default) relative (offset) absolute (pinned) position demo — see how the three behave differently

position: static (default)

Elements are placed in the normal document flow. The top / right / bottom / left properties have no effect.

position: relative

The element stays in the normal flow but you can nudge it with top / left / right / bottom. The original space is still reserved. Also creates a positioning context for absolutely-positioned children.

CSS — Relative Positioning
.nudge { position: relative; top: 10px; /* pushed down 10px */ left: 20px; /* pushed right 20px */ }

position: absolute

The element is removed from the normal flow and positioned relative to its nearest positioned ancestor (any ancestor with a position other than static). If none exists, it positions relative to the initial containing block (the viewport).

CSS — Absolute Positioning (common pattern)
/* Parent is the positioning context */ .card { position: relative; } /* Badge pinned to the top-right corner of .card */ .card .badge { position: absolute; top: -8px; right: -8px; } /* Centre an element inside its parent */ .centered { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

position: fixed

Like absolute but positioned relative to the viewport, and stays there even when the page scrolls. Used for sticky headers, floating action buttons, and cookie banners.

CSS — Fixed Positioning
.navbar { position: fixed; top: 0; left: 0; right: 0; /* or width: 100% */ z-index: 1000; background: white; box-shadow: 0 2px 8px rgba(0,0,0,0.1); }

position: sticky

The element behaves like relative until it reaches a specified scroll threshold, then acts like fixed. The element stays within its parent container.

CSS — Sticky Positioning
.section-header { position: sticky; top: 0; /* sticks when it reaches the top */ background: white; z-index: 10; padding: 12px 0; } /* Table header that sticks on scroll */ thead th { position: sticky; top: 0; }

The z-index Property

Controls the stacking order of positioned elements. Higher values appear on top. Only works on elements that have a position value other than static.

CSS — z-index Stack
.modal-overlay { z-index: 900; } .modal { z-index: 1000; } .tooltip { z-index: 1100; } /* Using CSS variables for a z-index scale */ :root { --z-nav: 100; --z-dropdown:200; --z-modal: 1000; --z-toast: 1100; }
overflow: hidden and Sticky

If any ancestor of a sticky element has overflow: hidden or overflow: auto, the sticky behaviour will break. This is a very common gotcha.

Wrapping Up

The display property determines how a box is generated; position determines where it lives. Together — block/inline flow, relative contexts, absolute pinning, fixed bars, and sticky headers — they give you complete control over where elements appear.
