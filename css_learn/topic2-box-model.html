The CSS Box Model

Every element on a web page is a rectangular box. The CSS Box Model describes how this box is built from four layers — content, padding, border, and margin — and understanding it is essential to mastering layout.

Content, Width & Height

By default, width and height set the size of the content area only. Padding and border are added on top of that.

CSS — Width and Height
.card { width: 300px; height: 200px; min-width: 200px; max-width: 100%; /* responsive cap */ }

Padding

Padding creates breathing room inside the element. It inherits the element's background color.

CSS — Padding Shorthand
/* All sides equal */ .box { padding: 20px; } /* Vertical (top/bottom) | Horizontal (left/right) */ .box { padding: 10px 24px; } /* Top | Horizontal | Bottom */ .box { padding: 10px 24px 16px; } /* Top | Right | Bottom | Left (clockwise) */ .box { padding: 10px 24px 16px 8px; } /* Individual sides */ .box { padding-top: 12px; padding-right: 20px; padding-bottom: 12px; padding-left: 20px; }

Border

The border sits between padding and margin. It has three sub-properties: width, style, and color.

CSS — Border
/* Shorthand: width style color */ .card { border: 2px solid #667eea; } /* Individual properties */ .card { border-width: 2px; border-style: dashed; /* solid | dashed | dotted | double | none */ border-color: #667eea; } /* Per-side */ .sidebar { border-left: 4px solid #10b981; } /* Rounded corners */ .pill { border-radius: 999px; } .card { border-radius: 12px; /* all corners */ border-radius: 8px 8px 0 0; /* top-left top-right bottom-right bottom-left */ }

Margin

Margin creates space outside the element. It is always transparent — backgrounds do not show through.

CSS — Margin & Centering
/* Same shorthand as padding */ .section { margin: 2em 0; } /* Center a block element horizontally */ .container { width: 800px; margin: 0 auto; /* top/bottom 0, left/right auto */ } /* Negative margin — pulls elements closer */ .overlap { margin-top: -20px; }
Margin Collapsing

When two vertical margins meet (e.g., a paragraph followed by another paragraph), they collapse into a single margin equal to the larger of the two. This only applies to block-level elements in normal flow — it does not happen with flex or grid children.

box-sizing: The Game Changer

The Default Problem

By default (box-sizing: content-box), if you set width: 300px; padding: 20px; border: 2px solid, the element is actually 344px wide (300 + 40 padding + 4 border). This makes sizing unpredictable.

CSS — box-sizing Fix
/* Modern reset — apply to every element */ *, *::before, *::after { box-sizing: border-box; } /* Now padding and border are INCLUDED in width/height */ .card { width: 300px; padding: 20px; border: 2px solid #667eea; /* Total rendered width = exactly 300px */ }
Always Use border-box

Add *, *::before, *::after { box-sizing: border-box; } at the top of every stylesheet. It is the first line in most professional CSS frameworks.

The outline Property

outline looks like a border but does not affect layout — it sits outside the border and takes up no space. Browsers use it for focus rings.

CSS — Outline
/* Accessible focus indicator — don't remove without a replacement */ button:focus { outline: 3px solid #667eea; outline-offset: 4px; } /* Custom focus style */ input:focus { outline: none; border: 2px solid #667eea; box-shadow: 0 0 0 3px rgba(102,126,234,0.3); }

Wrapping Up

The Box Model — content, padding, border, margin — governs every element's size and spacing. Setting box-sizing: border-box globally makes sizing intuitive and predictable.
