CSS Introduction & Selectors

CSS (Cascading Style Sheets) is the language that brings web pages to life. While HTML structures your content, CSS controls everything about how it looks — colors, fonts, spacing, layout, and animation. Mastering selectors is the first step to writing powerful CSS.

What is CSS?

CSS works by selecting HTML elements and applying declarations (property–value pairs) to them. Every CSS rule follows the same basic structure:

CSS — Anatomy of a Rule
/* selector { property: value; } */ h1 { color: #2c3e50; font-size: 2em; font-weight: 700; } .intro { background: #f0f4ff; padding: 20px; border-radius: 8px; }

Ways to Add CSS

Method Where Best For
External stylesheet <link rel="stylesheet" href="style.css"> Production — keeps HTML clean, styles shared
Internal / embedded <style> … </style> in <head> Single-page demos, email templates
Inline style="…" on the element One-off overrides, dynamic JS styling

Basic Selectors

1. Element (Type) Selector

Targets every instance of an HTML element.

CSS — Element Selector
p { color: #4b5563; line-height: 1.7; } h2 { font-size: 1.6em; margin-bottom: 0.5em; }

2. Class Selector

Targets any element that has the matching class attribute. Classes are reusable and the most common selector in real projects.

CSS — Class Selector
/* HTML: <div class="card"> */ .card { background: white; border-radius: 12px; box-shadow: 0 4px 16px rgba(0,0,0,0.1); padding: 24px; } /* Multiple classes on one element */ .btn.btn-primary { background: #667eea; color: white; }

3. ID Selector

Targets a single element with the matching id. IDs must be unique per page.

CSS — ID Selector
/* HTML: <nav id="main-nav"> */ #main-nav { position: sticky; top: 0; background: #1e293b; z-index: 1000; }
Prefer Classes Over IDs for Styling

IDs have very high specificity, making them hard to override later. Reserve IDs for JavaScript hooks or anchor links, and use classes for all your styling.

Intermediate Selectors

Attribute Selector

CSS — Attribute Selectors
/* Exact match */ input[type="email"] { border: 2px solid #667eea; } /* Starts with */ a[href^="https"] { color: #10b981; } /* Ends with */ a[href$=".pdf"] { color: #ef4444; } /* Contains */ a[href*="github"] { font-weight: bold; }

Grouping Selector

CSS — Grouping
/* Apply the same rule to multiple selectors */ h1, h2, h3, h4 { font-family: 'Georgia', serif; color: #1a202c; line-height: 1.3; }

Combinator Selectors

CombinatorSyntaxMeaning
Descendantdiv pAny <p> inside a <div>
Childul > liDirect children only
Adjacent siblingh2 + pThe first <p> directly after an <h2>
General siblingh2 ~ pAll <p> siblings after an <h2>
CSS — Combinators in Practice
/* Descendant: any <a> inside .nav */ .nav a { color: white; text-decoration: none; } /* Child: direct <li> children of .menu */ .menu > li { display: inline-block; } /* Adjacent: the <p> immediately after an <h2> */ h2 + p { font-size: 1.1em; color: #6b7280; }
The Universal Selector

The * selector matches every element on the page. It is commonly used in a CSS reset: * { box-sizing: border-box; margin: 0; padding: 0; }

Wrapping Up

You have learned how CSS rules are written, three methods to include CSS, and the core selector types — element, class, ID, attribute, grouping, and combinators. These tools let you target any element on a page with precision.
